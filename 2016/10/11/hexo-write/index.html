添加新文章

在网站根目录下，执行git bash命令创建一篇新文章
$ hexo new [layout] <title>

  • title指代是新文章名称。
  • layout: 布局，有三种 post,page和draft，
    默认的是post. 指代的是新建文章存储路径。
布局 路径
post source/_posts
page source
draft source/_drafts
  • page 在之前创建分类，标签等页面时使用过。可以看到新的文档是在source下创建新的目录，并创建一个index.md文章。
  • post 是在source/_posts下创建title文章。
  • draft 草稿。新建的文章存放在source/_drafts下，但不会显示在网站中。如果想发布这篇文章，利用下面命令：

文章属性

文章属性，这里指代的是文章的标识。在文章最上方以-– 分隔的区域。下面列出常用参数：

参数 描述
title 文章标题
date 建立时间
updated 更新时间
comments 文章评论功能
tags 标签
categories 分类
  • comments 默认true开启评论。false 关闭评论
  • 分类和标签的区别是：分类具有顺序性和层次性，而标签没有。

    举例来说：分类 A,B 表示分类A下面的小分类B。可能A下面还有其它分类。
    而标签 A,B 表示文章具有标签A和B.标签A和标签B是独立存在的。

表格支持

  • 注意事项：第二行表示表格格式。
    • 冒号在右边，表示右对齐.
    • 冒号在左边，表示左对齐。

待办事宜 Todo 列表

  • 使用带有[ ]或 [x]（未完成或已完成）项的列表语法撰写一个待办事宜列表，并且支持子列表嵌套以及混用Markdown语法

  • [ ] 今天工作

    • [ ] 写一篇文章
    • [ ] markdown语法自习
    • [x] 昨日工作总结
  • [ ] 明日外出准备
    • [x] 水杯
    • [x] 背包
    • [x] 眼镜
