hexo介绍

Node.js 是运行在服务端的 JavaScript。
一个基于Google的V8引擎，由事件驱动I/O服务端JavaScript环境。
而 Hexo 是在 Node.js上创建的快速、简洁且高效的博客框架，可将 Markdown 文件快速的生成静态网页。
如果你想管理自己的博客，不想花费太多的精力和金钱，又不想受其它平台的约束，hexo是一个很好的选择。
本文主要介绍在windows下安装的步骤和过程。

软件需求：

git: 2.9.3版本 http://git-scm.com/
node.js:6.5.0版本 https://nodejs.org
notepad++: https://notepad-plus-plus.org/

node包的管理

  • NPM介绍
    NPM（node package manager）介绍：
    NPM，node应用包管理器。主要功能管理node包，包括：安装、卸载、更新、查看、搜索、发布。
    NPM安装分为本地安装和全局安装。区别是，本地安装，只能在安装目录下运行，全局安装则是在其它目录下也能执行。
    本地安装： npm install pkg (pkg为安装包名)
    全局安装： npm install -g pkg

  • NPM源的切换
    Node.js 官方源默认是：http://registry.npmjs.org。在国内使用，将源替换为淘宝镜像源 http://npm.taobao.org/
    替换后，除了publish(发布),其它命令都可由cnpm代替npm执行。

安装之后，会在 E:/heox下生成博客目录和文件。此时本地博客安装完成。

目录简要说明

  • _config.yml 网站全局配置信息
  • scaffolds 模板文件夹。新建文章时，Hexo 会根据 scaffold 来建立文件。
  • themes 主题文件夹。

hexo根据全局配置文件，调用主题生成静态页面。

hexo常用命令

  • hexo clean 清除缓存文件和已生成的静态文件
  • hexo generate 生成静态文件。简写为 hexo g
  • hexo server 启动服务器。 简写为 hexo s ，默认端口4000.访问网址：http://localhost:4000/ 。重设为80端口参数 -p 80 调试参数：–debug
  • hexo deploy 部署上传网站。
  • hexo new 新建一篇title文章。默认使用 _config.yml 中的 default_layout 参数。如果title包含空格，用引号括起来。
