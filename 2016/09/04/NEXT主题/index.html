全局配置

在 _config.yml中可修改大部份的配置。

配置简要说明：(以我的网站为例)

域名绑定

　　域名绑定分为两个步骤：

  • 在你域名注册解析的网站中，将域名指向仓库名
  • 网站目录中添加CNAME文件，指定网站域名

    CNAME文件存放位置： 根目录下/source 文件夹中。假如网站目录 e:/hexo，那么存放位置是 e:/hexo/source/中
    CNAME文件内容直接填写绑定域名信息。其它不用写。比如我的网站： blog.kuxinhua.com

添加主题

  • hexo 主题比较多。这里为大家介绍几个网站

    • hexo-theme: https://hexo.io/themes/
    • hexo-github-theme-list: https://github.com/hexojs/hexo/wiki/Themes
    • 知乎上的讨论：http://www.zhihu.com/question/24422335

  • 我选择的是NEXT主题,比较干净纯洁。

下载启用主题

  • 使用git 克隆最新主题版本到 目录themes/ 中。

  • 启用主题

    • 打开站点 配置文件 ，修改theme字段值为：next

  • 验证主题

    启动本地Hexo站点，开启调试模式。命令是： hexo s –debug.
    使用浏览器访问 http://localhost:4000 检查站点是否正确运行。

主题设定

选择Scheme

next主题提供三种Scheme外观。在主题配置文件中(/themes/next/_config/yml)切换。

Muse:默认的scheme，next最初版本，黑白主调，大量留白
Mist:Muse的紧凑版本，整洁有序的单栏外观
Pisces - 双栏Scheme，小家碧玉似的清新

主题语言

  • 上节全局配置文件中说明中有设定。

    全局配置文件为：网站根目录下 _config.yml，不要和主题配置文件弄混淆。

  • 一些常用语言如下：

    english: en
    简体中文： zh-Hans
    繁体中文： zh-hk或是 zh-tw

设置菜单

  • 菜单栏就是主页的导航栏。
  • 菜单栏分为二个部分：第一菜单项（名称和链接），第二菜单项对应的图标和显示文本。

  • 菜单项对应的配置是主题配置文件中的menu字段。设置格式是：item name: link.如下所示

    • 这其中 ：分类页，标签页，关于页和404页面要手动创建这个页面。
      menu:
      home: / #next显示文本是：主页
      categories: /categories #分类页
      #about: /about #关于页面，前面加#表示不显示
      #archives: /archives #归档页
      tags: /tags #标签页
      #search: /
      #commonweal: /404.html #404页面
    • 手动创建页面
    • 以归档页为例,网站根目录下执行git bash命令：
      $ hexo new page "archives"
      INFO Created: E:\hexo\source\archives\index.md
    • 编辑生成的页面：

  • 菜单项文字和图标

    • 菜单项显示的对应文字放置在网站使用主题目录下的 languages/{language}.yml （{language} 是网站使用的语言）。如下所示：
      menu: home: 首页 archives: 归档 categories: 分类 tags: 标签 about: 关于 search: 搜索 commonweal: 公益404

    • 在NEXT主题中，显示的菜单图片是由Font Awesome提供的图片。

    • 在主题配置文件中设置：配置方式为 菜单项: 菜单图片名 。
      menu_icons: enable: true #启用图标 home: home about: user categories: th tags: tags archives: archive commonweal: heartbeat
